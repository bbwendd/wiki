# Confirm Plan 檢查表

Confirm Plan 是 Plan Mode 進入 Auto Mode 的硬性門檻。

## 必填項目

計畫中必須清楚說明：

- 目標：這次要完成什麼。
- 交付物：最後會產出哪些檔案、文件、訊息或結果。
- 資料來源：會使用哪些 raw、wiki、schema、外部來源或 MCP connector。
- 操作範圍：哪些會改、哪些不會改。
- 驗證方式：如何證明完成。
- 風險：可能出錯、缺資料、授權不足或需要使用者確認的地方。
- 不可交付條件：哪些情況下必須停止、退回或詢問。

## Confirm Plan 通過條件

以下全部成立才算通過：

- 目標可執行且沒有重大模糊。
- 交付物明確。
- 資料來源可追溯。
- 驗證方式具體。
- 對外行動與高風險操作已標示。
- 使用者已用明確文字確認。

## 不可進入 Auto Mode 的情況

- 使用者尚未明確確認。
- 交付物或驗證方式不清楚。
- 需要權限但尚未取得。
- 會涉及對外承諾但未列出確認點。
- 任務資料不足且不足以安全規劃。

## 簡短任務格式

小任務仍需最小計畫：

```md
## Plan

- 目標：
- 交付物：
- 驗證：
- 風險 / 待確認：

請回覆「確認計畫」後，我再執行。
```
