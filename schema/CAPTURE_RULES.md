# 資料捕捉規則

## 放進 raw 的資料

- Email 原文或寄件備份。
- 會議逐字稿、會議紀錄、錄音摘要。
- 專案附件、規格、計畫書、簡報。
- 長官、業主、廠商、合作夥伴的明確指示。
- 使用者口述的背景、偏好、決策。

## raw 檔名規則

建議格式：

`YYYY-MM-DD_來源_主題.md`

範例：

- `2026-06-27_Gmail_某專案需求確認.md`
- `2026-06-27_會議紀錄_主權AI工作會議.md`
- `2026-06-27_口述_工作助理需求.md`

## raw 內容格式

```md
# 標題

- 日期：
- 來源：
- 相關專案：
- 相關人物：
- 狀態：

## 原始內容

貼上原文、摘要或來源描述。

## 可抽取事項

- 事實：
- 待辦：
- 風險：
- 待確認：
```

## 整理到 wiki 的條件

只要資料符合以下任一條件，就應該整理到 `wiki`：

- 未來會再次查找。
- 影響專案進度、承諾、風險或產出。
- 涉及重要人物、窗口或合作關係。
- 可以變成固定流程、模板或檢查表。
