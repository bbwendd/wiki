# 工具使用規則

本檔定義各類工具的使用邊界。若工具不可用，先使用可替代的輸入方式，例如貼上文字、上傳檔案、截圖或手動提供連結。

## Gmail / Email

- 可做：搜尋、閱讀、摘要、判斷是否需回覆、建立回覆草稿、整理等待對方。
- 不可做：未經確認直接寄出、承諾交期或條件。
- 必須保留：寄件人、收件人、日期、主旨、thread 或來源描述。
- 若使用者說已寄出或已收到新回覆，先以 Gmail 實際內容為準。

## Line 訊息

- 若無 Line connector，使用者需提供文字、截圖或匯出內容。
- 可做：判斷語氣、整理重點、產生回覆草稿。
- 不可做：未經確認自動傳送。

## Google Calendar / 行程

- 可做：整理會議、找空檔、產生排程建議、建立會議草案。
- 不可做：未經確認建立、修改、取消會議。
- 必須標明：時區、日期、開始結束時間、參與者、待確認事項。

## Google Drive / 文件來源

- 可做：搜尋、讀取、摘要、匯出、整理文件來源。
- 不可做：未經確認刪除、移動或改名重要文件。
- 必須保留：檔名、位置、版本、來源日期。

## 文件 / 簡報 / 表格

- Word / DOCX：用於正式計畫書、活動企劃、投標文件、會議紀錄。
- PowerPoint / Slides：用於簡報、論壇成果發表、專案簡報。
- Excel / Sheets：用於追蹤表、預算表、名單表、時程表。
- 產出前需確認：對象、用途、格式、欄位或章節。

## 網路搜尋

- 可做：查找專案專業資料、政府標案資料、背景案例。
- 必須做：附來源連結、區分事實與推論、標示查詢日期。
- 不可做：把未驗證內容寫成官方事實。

## Canva / 圖像生成 / 美編

- 可做：DM brief、版型方向、文案、視覺素材需求、Canva 設計草案。
- 不可做：未確認品牌規範時假設品牌色、logo 或正式版權素材。
- 必須確認：尺寸、用途、活動名稱、主視覺、輸出格式。

## 音檔與逐字稿

- 可做：根據可讀音檔或逐字稿整理會議紀錄、決議、待辦。
- 若無法直接轉錄：要求使用者提供逐字稿或可用轉錄輸出。
- 必須標示：轉錄是否完整、聽不清楚處、待確認內容。
